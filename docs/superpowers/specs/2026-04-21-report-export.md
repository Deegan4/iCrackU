# Sub-project 3: Report Export

**Date:** 2026-04-21
**Status:** Approved
**Depends on:** Sub-project 1 (result JSON structure), Sub-project 2 (profile JSON structure)

---

## Goal

Generate HTML and/or Markdown reports from any saved result JSON (single lookup or profile). Triggered via `--report <file>` CLI flag, as a post-lookup `--report` flag, or from the interactive menu.

---

## Section 1: Architecture ‚Äî `core/reporter.py`

Two public functions:

```python
def generate_html(data: dict) -> str: ...
def generate_markdown(data: dict) -> str: ...
```

Both accept the standard result JSON structure written by `core/saver.py`:
```json
{
  "timestamp": "...",
  "type": "email",
  "query": "foo@bar.com",
  "tools": [{ "name": "holehe", "returncode": 0, "output": "..." }]
}
```

Also accepts the profile JSON from Sub-project 2 (detected via `data["type"] == "profile"`), iterating over `data["targets"]` to render each lookup section.

Reports saved alongside source JSON:
```
results/
  2026-04-21_063340_email_foo.json
  2026-04-21_063340_email_foo.html
  2026-04-21_063340_email_foo.md
```

---

## Section 2: CLI Wiring

**Standalone report generation:**
```bash
iCrackU --report results/2026-04-21_email_foo.json
iCrackU --report results/profile_2026-04-21_johndoe.json --format both
```

**Post-lookup flag** (combined with any lookup flag):
```bash
iCrackU --email foo@bar.com --report
iCrackU --username johndoe --report --format md
```

**`--format` flag:** `html` (default) | `md` | `both`

**Menu option:** `R  Generate report` ‚Äî shows saved results list, user picks number, selects format, report generated and path printed.

---

## Section 3: Report Content

### HTML Report

Self-contained dark-theme file (inline CSS only, no external dependencies, works offline).

Structure:
- **Header:** iCrackU ASCII logo (text, not image) + timestamp + lookup type + query/target
- **Summary table:** tool | status | lines
- **Per-tool sections:** `<details><summary>toolname</summary><pre>output</pre></details>` ‚Äî collapsible
- **Footer:** generated by iCrackU + timestamp

Dark theme palette: `#0d0d0d` background, `#00ff41` accent (Mr. Robot green), `#cccccc` body text, `#1a1a1a` section backgrounds.

### Markdown Report

GitHub-renderable `.md`:

```markdown
# iCrackU Report ‚Äî email ‚Äî foo@bar.com
**Generated:** 2026-04-21 06:33:40

## Summary
| Tool | Status | Lines |
|---|---|---|
| holehe | ‚úì | 18 |
| theHarvester | ‚úì | 42 |

## holehe
\`\`\`
[raw tool output]
\`\`\`

## theHarvester
\`\`\`
[raw tool output]
\`\`\`
```

## Section 4: Testing

**`tests/test_reporter.py`** ‚Äî new file:

- `test_generate_html_contains_query` ‚Äî assert query string appears in HTML output
- `test_generate_html_contains_tool_name` ‚Äî assert tool name appears in HTML output
- `test_generate_html_is_valid_structure` ‚Äî assert `<html>`, `<body>`, `<details>` present
- `test_generate_markdown_contains_summary_table` ‚Äî assert `| Tool |` header present
- `test_generate_markdown_contains_tool_section` ‚Äî assert `## holehe` heading present
- `test_generate_markdown_contains_output` ‚Äî assert tool output text present
- `test_reporter_handles_profile_json` ‚Äî pass profile-type data dict, assert all target sections rendered

No file I/O in reporter tests ‚Äî test the string output of `generate_html` and `generate_markdown` directly.
